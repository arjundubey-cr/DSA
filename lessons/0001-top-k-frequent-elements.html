347. Top K Frequent Elements

Medium Bucket Sort Hash Map O(n) time

The Problem

Problem Statement

Given an integer array nums and an integer k, return the k most frequent elements. You may return the answer in any order.

Example 1
Input: nums = [1,1,1,2,2,3], k = 2
Output: [1, 2]
Example 2
Input: nums = [1], k = 1
Output: [1]

Constraints: 1 ≤ nums.length ≤ 105  ·  Answer is guaranteed unique  ·  Follow-up: must be better than O(n log n)

Step 1 — The Obvious Approach

You already know how to count frequencies (you did it in Valid Anagram and Group Anagrams). So the natural first instinct is:

  1. Build a frequency map with a dict
  2. Sort by frequency descending
  3. Return the first k keys
def topKFrequent(self, nums, k):
    count = {}
    for n in nums:
        count[n] = count.get(n, 0) + 1
    # sort keys by frequency, highest first
    return sorted(count, key=lambda x: count[x], reverse=True)[:k]

This works. But the follow-up constraint says: do better than O(n log n). The sort is the bottleneck.

ApproachTimeSpace
Sort by freqO(n log n)O(n)
Min-heapO(n log k)O(n + k)
Bucket sort ✓O(n)O(n)

Step 2 — The Key Insight

Core Insight
Frequency is bounded by n. If the array has n elements, no element can appear more than n times. This means we can use the frequency as an array index — no sort needed.

Create an array freq of size n + 1, where freq[i] stores all elements that appear exactly i times.

Then scan from the right (highest frequency) and collect elements until you have k. Done.

Step 4 — The Algorithm

def topKFrequent(self, nums, k):
    count = {}
    freq = [[] for _ in range(len(nums) + 1)]  # ①

    for n in nums:
        count[n] = 1 + count.get(n, 0)          # ②

    for n, c in count.items():
        freq[c].append(n)                          # ③

    res = []
    for i in range(len(freq) - 1, 0, -1):       # ④
        for n in freq[i]:
            res.append(n)
            if len(res) == k:
                return res
  • ①Allocate n+1 buckets. freq[0] unused (no element appears 0 times). Max frequency is n, so freq[n] is the last valid bucket.
  • ②Standard frequency count — same pattern from Valid Anagram.
  • ③Place each element into the bucket matching its count. Element n with count c goes into freq[c].
  • ④Iterate from high to low frequency, collecting elements until we have k. Early return as soon as we do — no wasted work.
Why This Is O(n)
No sort. The frequency count pass is O(n). Building freq[] is O(n). The collection pass visits at most n elements total across all buckets. Every step is linear.
