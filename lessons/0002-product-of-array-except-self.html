238. Product of Array Except Self

Medium Prefix Product Postfix Product O(n) · No division

The Problem

Problem Statement

Given an integer array nums, return an array output such that output[i] is equal to the product of all elements of nums except nums[i]. Must run in O(n) time. No division.

Example 1
Input: [1, 2, 3, 4]
Output: [24, 12, 8, 6]
Example 2
Input: [-1, 1, 0, -3, 3]
Output: [0, 0, 9, 0, 0]

Constraints: 2 ≤ nums.length ≤ 105  ·  Follow-up: O(1) extra space (output array doesn't count)

Step 1 — Why Can't We Just Divide?

Instinct: compute total product, then output[i] = total / nums[i]. Two reasons this breaks:

  1. Division by zero — any zero in nums kills it
  2. Constraint — problem explicitly forbids division

So we need a different way to express "product of everything except i".

Step 2 — The Insight

Core Insight
For any index i:

output[i] = (product of everything LEFT of i) × (product of everything RIGHT of i)

We can compute both halves in two passes. No division needed.

Step 3 — See It

For nums = [1, 2, 3, 4]:

i=0i=1i=2i=3
nums 1 2 3 4
LEFT product 1 1 1×2=2 1×2×3=6
RIGHT product 2×3×4=24 3×4=12 4 1
output = L × R 1×24=24 1×12=12 2×4=8 6×1=6

LEFT at i=0 is 1 (nothing to the left — empty product). RIGHT at i=3 is 1 (nothing to the right — empty product).

Step 4 — The Algorithm

Naive implementation: build two full arrays, then multiply. Works but uses O(n) extra space.

# Naive — O(n) space (two extra arrays)
def productExceptSelf(self, nums):
    n = len(nums)
    left  = [1] * n
    right = [1] * n

    for i in range(1, n):
        left[i] = left[i-1] * nums[i-1]
    for i in range(n-2, -1, -1):
        right[i] = right[i+1] * nums[i+1]

    return [left[i] * right[i] for i in range(n)]

Optimize: skip the left[] and right[] arrays. Use the output array itself plus a running variable.

# Optimal — O(1) extra space
def productExceptSelf(self, nums):
    n = len(nums)
    output = [1] * n          # ①

    prefix = 1
    for i in range(n):       # ②
        output[i] = prefix
        prefix *= nums[i]

    postfix = 1
    for i in range(n-1, -1, -1):  # ③
        output[i] *= postfix
        postfix *= nums[i]

    return output
  • ①Output starts as all 1s — neutral for multiplication. We fill left products in pass ②, then multiply by right products in pass ③.
  • ②Left pass. Before touching index i, prefix holds the product of nums[0..i-1]. Write it to output[i], then update prefix. At i=0, prefix is 1 (empty product). Correct.
  • ③Right pass — reverse. Before touching index i, postfix holds the product of nums[i+1..n-1]. Multiply into output[i] (which already holds the left product). Then update postfix.
Why Two Passes?
Each pass runs the loop once — O(n). Two passes = 2×O(n) = O(n). No nested loops, no sorting. The prefix and postfix variables are O(1) extra space; only the output array scales with input.
