Encode & Decode Strings

Medium Length-Prefix Encoding String Parsing

The Problem

Problem Statement

Design an algorithm to encode a list of strings into a single string, and decode that single string back into the original list. The encoded string may be transmitted over a network where any character could appear in the original strings.

Implement two functions:

  • encode(strs: list[str]) -> str
  • decode(s: str) -> list[str]
Example
Input: ["neet", "code", "love", "you"]
Encoded: "4#neet4#code4#love3#you"
Decoded: ["neet", "code", "love", "you"]

Constraints: strings can contain any character — including #, commas, spaces, newlines.

Step 1 — Why a Simple Delimiter Fails

First instinct: join with a delimiter like "," or "#", then split on decode.

Problem
What if a string contains your delimiter?

encode(["a,b", "c"]) → "a,b,c"
decode("a,b,c") → ["a", "b", "c"]  ← wrong. Should be ["a,b", "c"].

Any single fixed delimiter breaks the moment a string contains that character. Since strings can contain any character, no single delimiter is safe.

Step 2 — The Insight: Length-Prefix Encoding

Core Insight
Instead of a delimiter between strings, store the length before each string.

Format: {len}#{content} — repeated for each string.

On decode, read the length, skip #, then slice exactly that many characters. No ambiguity. No parsing of the content.

The # here is a separator between the length and the content — not a delimiter between strings. Even if the content contains #, we never look inside it. We consume exactly len characters and move on.

Step 3 — See the Encoding

For ["neet", "code", "love", "you"]:

Encoded string — character by character
"neet"
4 # neet
"code"
4 # code
"love"
4 # love
"you"
3 # you
length # separator string content

Full encoded string: "4#neet4#code4#love3#you"

Step 4 — Decoding: Read the Length, Slice, Advance

Pointer starts at i = 0. Each iteration:

  1. Find the # starting from i
  2. Parse the integer before it — that's the string length n
  3. Slice n characters after the #
  4. Advance i past what we consumed
i=0
Find # at j=1. Length = s[0:1] = "4" → n=4
Slice s[2 : 2+4] = "neet" → append.
Advance i = 2+4 = 6
i=6
Find # at j=7. Length = s[6:7] = "4" → n=4
Slice s[8 : 8+4] = "code" → append.
Advance i = 8+4 = 12
i=12
Find # at j=13. Length = "4" → n=4. Slice → "love".
Advance i = 18
i=18
Find # at j=19. Length = "3" → n=3. Slice → "you".
Advance i = 22. i = len(s). Done.

Step 5 — The Code

class Solution:

    def encode(self, strs: list[str]) -> str:
        res = ""
        for s in strs:
            res += f"{len(s)}#{s}"       # ①
        return res

    def decode(self, s: str) -> list[str]:
        res, i = [], 0
        while i < len(s):
            j = s.index("#", i)        # ②
            n = int(s[i:j])             # ③
            res.append(s[j+1 : j+1+n]) # ④
            i = j + 1 + n             # ⑤
        return res
  • ①Each string becomes {length}#{content}. No separator needed between pairs — the length tells us exactly where each string ends.
  • ②s.index("#", i) finds the next # starting from i. Crucial: we search from i, not 0, so we don't find a # inside a previous string's content.
  • ③s[i:j] is the length digits. Convert to int.
  • ④Content starts one position after # (at j+1) and is exactly n chars long. Slice precisely — no scanning, no splitting.
  • ⑤Advance i past the chunk we just consumed: the # position + 1 (for # itself) + n (for the content).
Why This Is Robust
We never look inside the content for delimiters. We read the length, then blindly slice that many characters. A string containing "3#hi" or "###" or "," is handled perfectly — we skip right over it.

Time: O(n) encode, O(n) decode. Space: O(n) for the encoded string and result list.

Step 6 — Edge Cases to Consider

  • Empty string in list: "" → encodes as "0#". Decodes back to "". Works.
  • String containing #: "a#b" → encodes as "3#a#b". Decode finds first # at j=1 via index("#", i), reads length 3, slices "a#b". Works.
  • Empty list: encode returns "", decode of "" returns [] (while loop never runs). Works.
  • Multi-digit lengths: "hello world" is 11 chars → "11#hello world". index("#") finds the first # correctly, so s[i:j] = "11". Works.
